## Анализ результатов A/B-теста для дейтингового приложения
**Стек:** *Python, Pandas, NumPy, SciPy, Seaborn, Matplotlib, Jupyter Notebook, Git, математическая статистика.*

### Бизнес-задача:
Компания протестировала изменение стоимости премиум-подписки при оплате через две новые платёжные системы. Необходимо было оценить, влияет ли новая цена на метрики, и решить, стоит ли внедрять изменение для всех пользователей.
### Данные:
Для исследования предоставлены данные по трём группам пользователей: *test, control_1 и control_2.*
Для каждой группы были доступны два датасета:
* информация о пользователях (`uid`, `age`, `country`, `gender` и др. характеристики);
* информация о транзакциях (`uid`, `country`, `joined_at`, `paid_at`, `revenue` и др.).
### Ход работы:
* Провела исследование и предобработку данных: проверила корректность групп, обработала дубликаты и пропуски.
* Сравнила контрольные группы (**А/А-тест**) и оценила корректность формирования тестовой выборки.
* Рассчитала ключевые метрики: **ARPU**, **ARPPU**, **конверсию в премиум**.
* Применила статистические тесты (**Манна–Уитни**, **χ²**, **Bootstrap**) для проверки семи гипотез.
### Результат:
Изменение стоимости не привело к росту выручки и конверсии. Увеличение ARPPU оказалось статистически значимым, но не являлось целевой метрикой.
### Рекомендация:
Не внедрять изменение цены для всех пользователей, оставить текущую стратегию монетизации.
